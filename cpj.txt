Problem Statement:  Tiered Discount Calculator  
A shopping website offers discounts based on the total purchase amount  as follows:  
 If total amount < ₹1000 → No discount  
 If total amount ≥ ₹1000 and < ₹5000 → 10% discount  
 If total amount ≥ ₹5000 and < ₹10000 → 20% discount  
 If total amount ≥ ₹10000 → 25% d iscount + ₹500 flat discount  
Task  
Write a program that:  
1. Accepts the total purchase amount  as input.  
2. Applies the appropriate discount using nested conditional logic . 
3. Displays the final payable amount . 
Input Format  
 A single integer representing the total purchase amount.  
Output Format  
 Print the final amount after applying the discount.  
Sample Input  
12000  
 
Sample Output  
Final amount to be paid:  8500

Java Solution:
import java.util.Scanner;  
 
public class DiscountCalculator  { 
    public static void main(String[] args)  { 
        Scanner sc = new Scanner(System.in);  
        System.out.print( "Enter total purchase amount: " ); 
        int amount = sc.nextInt();  
        double finalAmount;  
 
        if (amount < 1000) { 
            finalAmount = amount;  
        } else if (amount < 5000) { 
            finalAmount = amount - (amount * 0.10); 
        } else if (amount < 10000) { 
            finalAmount = amount - (amount * 0.20); 
        } else { 
            finalAmount = amount - (amount * 0.25) - 500; 
        } 
 
        System.out.println( "Final amount to be paid: "  + (int)finalAmount);

    } 
}

Problem Statement:  Calculate Final Order Cost with Multiple Discounts  
Description:  
You are designing a checkout system for an online store. Each order can contain multiple 
items. The system applies  discounts based on the total purchase amount  as well as item-
speci fic discounts . 
Discount rules:  
1. Item -level discount:  
o Some items have a fixed percentage discount.  
o Example: If an item costs $200 and has a 10% discount → item cost = 200 - 
20 = 180.  
2. Order -level discount:  
o After summing all item costs (after item -level discounts):  
 If total > $500, apply 10% off.  
 If total > $1000, apply a fixed discount of $150.  
o Only the discount that gives the maximum savings  is applied.  
Your task is to calculate the final order cost  after applying both item -level and order -level 
discounts.  
Input Format:  
 First line: integer n → number of items in the order.  
 Next n lines: two integers per line: price  discount_percent  (0 if no item -level 
discount).  
Output Format:  
 Print a single integer:  the final cost after all discounts.  
Constraints:  
 
Sample Input:  
3 
200 10  
400 0  
600 5  
Sample Output:  

1000 
Explanation  
1. Apply item -level discounts:  
o Item 1: 200 - 10% = 180  
o Item 2: 400 - 0% = 400  
o Item 3: 600 - 5% = 570  
Subtotal = 180 + 400 + 570 = 1150  
2. Apply  order -level discount:  
o 10% of 1150 = 115  
o Fixed discount = 150  
o Maximum discount = 150  
3. Final cost = 1150 - 150 = 1000

Java Solution:
import java.util.Scanner;  
 
public class FinalOrderCost  { 
    public static void main(String[] args)  { 
        Scanner sc = new Scanner(System.in);  
        int n = sc.nextInt();  
        double subtotal  = 0; 
         
        for(int i=0; i<n; i++){  
            int price = sc.nextInt();  
            int discountPercent  = sc.nextInt();  
            subtotal += price - (price * discountPercent / 100.0); 
        } 
        sc.close();  
         
        double percentageDiscount  = 0; 
        int fixedDiscount  = 0; 
         
        if(subtotal > 500) percentageDiscount = subtotal * 0.10; 
        if(subtotal > 1000) fixedDiscount = 150; 
         
        double discount  = Math.max(percentageDiscount, fixedDiscount);  
        int finalCost  = (int)(subtotal - discount);  
        System.out.println(finalCost);  
    } 
}

Explanation:  
 Sum = 10+20+30+40+50 = 150

Java Solution:
import java.util.Scanner;  
 
public class SumModulo  { 
    public static void main(String[] args)  { 
        Scanner sc = new Scanner(System.in);  
        int n = sc.nextInt();  
        long m = sc.nextLong();  
        long total = 0; 
         
        for(int i = 0; i < n; i++){  
            long num = sc.nextLong();  
            total = (total + (num % m)) % m;  
        } 
        sc.close();  
         
        System.out.println(total);  
    } 
}

Difficulty:  Medium  
Description:  
Given three integers a, m, and p, compute  
(am) mod p  
You must ensure that intermediate results do not cause integer overflow . 
Input Format:  
 Single line containing three integers: a m p  
Output Format:  
 Single integer → (a^m) % p  
Constraints:  
 
Sample Input 1 : 
2 10 1000  
Sample Output 1:

Explanation:  
 210=1024  
 1024%1000 =24 
Note:  Use modular exponentiation  to avoid large intermediate results.

Java Solution:
import java.util.Scanner;  
 
public class ModularExponentiation  { 
    public static long modExp(long a, long m, long p){ 
        long result = 1; 
        a = a % p;  
        while(m > 0){ 
            if((m & 1) == 1){ 
                result = (result * a) % p;  
            } 
            a = (a * a) % p;  
            m >>= 1; 
        } 
        return result; 
    } 
     
    public static void main(String[] args)  { 
        Scanner sc = new Scanner(System.in);  
        long a = sc.nextLong();  
        long m = sc.nextLong();  
        long p = sc.nextLong();  
        sc.close();  
         
        System.out.println(modExp(a, m, p));  
    } 
}

approach  without directly calculating large factorials.

 Print the number of possible selections (combinations)  
Constraints

Java Solution:
import java.util.Scanner;  
 
public class Combinations  { 
    public static long combination (int n, int k) { 
        if (k > n) return 0; 
        k = Math.min(k, n - k); 
        long result = 1; 
 
        for (int i = 1; i <= k; i++) {  
            result = result * (n - i + 1) / i; 
        } 
        return result; 
    } 
 
    public static void main(String[] args)  { 
        Scanner sc = new Scanner(System.in);  
        int n = sc.nextInt();  
        int k = sc.nextInt();  
        sc.close();  
 
        System.out.println(combination(n, k));

public class AccessAudit  {

Problem Statement:  Store, Iterate, and Calculate Total Product Prices Using Arrays or Lists  
Problem Description  
In a retail billing system, the prices of multiple products need to be stored and processed. The 
task is to store a collection of numeric price values using a bas ic data structure such as an 
array (in C/Java) or a list (in Python), iterate over the collection to display each price, and 
also calculate and display the total price  of all products.  
Input Format  
• First line: integer N (number of products)  
• Second line : N numeric values (prices of products)  
Output Format  
• Print all the product prices in the same order they were entered  
• Print the total price of all products  
Sample Input  
5 
120 250 99 180 75  
Sample Output  
120 250 99 180 75  
Total Price = 724  
Explanation  
• The program reads N = 5  
• It accepts 5 product prices: 120 250 99 180 75  
• The prices are stored in an array/list  
• The program iterates over the collection to print each price  
• While iterating, it also adds each price to compute the total  
• The final t otal price is printed

Java Solution:
import java.util.Scanner;  
 
public class StorePrices {  
    public static void main(String[] args) {  
        Scanner sc = new Scanner(System.in);  
 
        int n = sc.nextInt();  
        int[] prices = new int[n];  
        int total = 0;  
 
        for (int i = 0; i < n; i++) {  
            prices[i] = sc.nextInt();  
            total += prices[i];  
        } 
 
        for (int i = 0; i < n; i++) {  
            System.out.print(prices[i] + " ");  
        } 
 
        System.out.println();

    }

Problem Statement:  Calculate Total Cost Using Product Prices and Quantities  
Problem Description  
In a billing or inventory system, each product has a price  and a quantity purchased . The 
task is to accurately calculate the total bill amount  by multiplying the price and quantity of 
each product and summing up all the individual costs. The pr ogram should store prices and 
quantities using arrays or lists and use basic arithmetic operations to compute the final total.  
Input Format  
• First line: integer N (number of different products)  
• Second line: N integers (prices of products)  
• Third line: N integers (quantities of products)  
Output Format  
• Print a single integer representing the total bill amount  
Sample Input  
4 
100 50 20 10  
2 5 3 4  
Sample Output  
420 
Explanation  
• The program reads N = 4  
• Product details:  
 Product 1: price 100, quantity 2 → cost = 200  
 Product 2: price 50, quantity 5 → cost = 250  
 Product 3: price 20, quantity 3 → cost = 60  
 Product 4: price 10, quantity 4 → cost = 40  
• Total cost = 200 + 250 + 60 + 40 = 550

Java Solution:
import java.util.Scanner;  
 
public class TotalBill  { 
    public static void main(String[] args)  { 
        Scanner sc = new Scanner(System.in);

            prices[i] = sc.nextInt();  
        } 
 
        for (int i = 0; i < n; i++) {  
            qty[i] = sc.nextInt();

Problem Statement: Simulate Car Movements Using the Two -Pointer 
Technique  
Problem Description  
In a straight road simulation, cars are positioned at different distances from a checkpoint. 
Some cars move from the left side  and others from the right side  toward the checkpoint. To 
efficiently analyze  their movement without repeatedly scanning the array, the Two -Pointer 
technique  is used. This technique allows simultaneous traversal from both ends of the array, 
reducing unnecessary iterations and improving performance.  
Input Format  
• First line: intege r N (number of cars)  
• Second line: N integers representing distances of cars from the checkpoint (sorted in 
increasing order)  
Output Format  
• Print the order in which cars are processed using two pointers (left and right)  
Sample Input  
6 
2 5 8 12 15 20  
Sample Output  
2 20 5 15 8 12  
Explanation  
• Cars are positioned at distances: 2 5 8 12 15 20  
• One pointer starts from the left (closest car)  
• Another pointer starts from the right (farthest car)  
• Cars are processed alternately from left and right  
• This sim ulates cars approaching from both directions efficiently

Java Solution:
import java.util.Scanner;  
 
public class TwoPointerCarSimulation  { 
    public static void main(String[] args)  { 
        Scanner sc = new Scanner(System.in);  
 
        int n = sc.nextInt();  
        int[] dist = new int[n]; 
 
        for (int i = 0; i < n; i++) {  
            dist[i] = sc.nextInt();  
        } 
 
        int left = 0, right = n - 1; 
 
        while (left <= right) {  
            if (left == right) {  
                System.out.print(dist[left] + " "); 
            } else { 
                System.out.print(dist[left] + " " + dist[right] + " "); 
            } 
            left++; 
            right--; 
        } 
 
        sc.close();  
    } 
}

Problem Statement:  Simulate Car Movement Using Two -Pointer Technique to 
Find Minimum Time  
Problem Description  
On a straight road, multiple cars are moving toward a checkpoint. Each car has a length  and a 
speed . A car fully passes the checkpoint only after its entire length  crosses the point.  
To efficiently simulate this process, cars are considered from both ends of the road  using the 
Two -Pointer technique . The goal is to determine the minimum  time required for all cars 
to completely pass the checkpoint , considering their speeds and lengths.  
The slowest effective movement (based on time taken) determines the final answer.  
Input Format  
• First line: integer N (number of cars)  
• Second line: N in tegers (lengths of the cars)  
• Third line: N integers (speeds of the cars)  
Output Format  
• Print the minimum time required for all cars to pass the checkpoint (in seconds)  
Sample Input  
4 
4 2 6 3  
2 3 1 2  
Sample Output  
6 
Explanation  
• There are 4 cars  
• Car lengths: 4, 2, 6, 3  
• Car speeds: 2, 3, 1, 2  
• Time taken by each car = length / speed  
• Times:  
– Car 1: 4 / 2 = 2  
– Car 2: 2 / 3 ≈ 0.67  
– Car 3: 6 / 1 = 6  
– Car 4: 3 / 2 = 1.5  
• The maximum time is 6  
• Hence, minimum time for all cars to pass is 6 seconds

Java Solution:
import java.util.Scanner;  
 
public class CarTwoPointer  { 
    public static void main(String[] args)  { 
        Scanner sc = new Scanner(System.in);  
 
        int n = sc.nextInt();  
        int[] length = new int[n]; 
        int[] speed = new int[n]; 
 
        for (int i = 0; i < n; i++)  
            length[i] = sc.nextInt();  
 
        for (int i = 0; i < n; i++)  
            speed[i] = sc.nextInt();  
 
        int left = 0, right = n - 1; 
        double maxTime = 0; 
 
        while (left <= right) {  
            double t1 = (double) length[left] / speed[left];  
            double t2 = (double) length[ right] / speed[right];  
 
            maxTime = Math.max(maxTime, Math.max(t1, t2));  
 
            left++; 
            right--; 
        } 
 
        System.out.println(( int) maxTime);  
        sc.close();  
    } 
}

Problem Statement: Brute -Force String Matching for Pattern 
Detection  
Problem Description  
In text processing applications such as search engines, plagiarism detection, and log analysis, 
it is often required to check whether a pattern string  appears within a text string . The 
brute -force string matching  approach is the simplest method, where the pattern is checked 
against every possible position in the text character by character. This problem helps in 
understanding how brute -force matching works and how its time complexity behaves in the 
worst case . 
Input Format  
• First line: a string T (the text ) 
• Second line: a string P (the pattern to search)  
Output Format  
• Print “Pattern Found” if P exists in T  
• Print “Pattern Not Found” if P does not exist in T  
Sample Input  
ABABABC  
ABABC  
Sample Output  
Pattern Found  
Explanation  
• The text string is: ABABABC  
• The pattern string is: ABABC  
• The algorithm starts matching from index 0  
• When a mismatch occurs, it shifts the pattern by one position  
• Eventually, the pattern fully matches starting at index 2  
• Hence, the pattern is found

Java Solution:
import java.util.Scanner;  
 
public class BruteForceStringMatch  { 
    public static void main(String[] args)  { 
        Scanner sc = new Scanner(System.in);  
 
        String text = sc.next();  
        String pattern = sc.next();  
 
        int n = text.length();  
        int m = pattern.length();  
        boolean found = false; 
 
        for (int i = 0; i <= n - m; i++) {  
            int j = 0; 
            while (j < m && text.charAt(i + j) == pattern.charAt(j)) {  
                j++; 
            } 
            if (j == m) {  
                found = true; 
                break; 
            } 
        } 
 
        if (found) 
            System.out.println( "Pattern Found" ); 
        else 
            System.out.println( "Pattern Not Found" ); 
 
        sc.close();  
    } 
}

import java.util.Scanner;  
 
public class CountSubstring  {

Problem Statement:  Locate All Occurrences of a Pattern Using 
KMP String Matching  
 
Problem Description  
In large text documents such as log files, DNA sequences, or articles, it is often required to 
find all positions where a given pattern occurs. The Knuth –Morris –Pratt (KMP) algorithm 
efficiently performs this task by preprocessing the pattern using the LPS (Longest Proper 
Prefix which is also Suffix) array and then scanning the text without re -checking previously 
matched characters. This approach ensures efficient searching even for large inputs.  
Input Format  
• First line: a string T (the text)  
• Second line: a string P (the pattern)  
Output Format  
• Print all startin g indices where the pattern occurs in the text  
• If no occurrence exists, print “Pattern Not Found”  
Sample Input  
ABABDABACDABABCABAB  
ABABCABAB  
Sample Output  
10 
Explanation  
• The text is ABABDABACDABABCABAB  
• The pattern is ABABCABAB  
• The pattern occurs st arting at index 10  
• KMP uses the LPS array to skip unnecessary comparisons  
Key Idea of KMP  
• Preprocess the pattern to build the LPS array  
• Use the LPS array to skip redundant comparisons  
• Never move the text pointer backward  
• Guarantees linear time co mplexity  
Algorithm (KMP Matching – High Level)  
Step 1: Read the text T and pattern P  
Step 2: Construct the LPS array for the pattern  
Step 3: Initialize two indices i (text) and j (pattern)  
Step 4: Compare T[i] and P[j]  
Step 5: If characters match, incremen t both i and j  
Step 6: If j reaches pattern length, report a match at (i − j) and update j using LPS  
Step 7: If mismatch occurs, update j using LPS or move i forward  
Step 8: Continue until the entire text is processed

            } else { 
                if (len != 0) 
                    len = lps[len - 1]; 
                else { 
                    lps[i] = 0; 
                    i++; 
                } 
            } 
        } 
    }

6

Highest Bid: 5100  
Next Highest Bid: 4500  
Explanation

Key Idea of Priority  Queue (Max -Heap)  
• Elements are stored in a heap structure  
• Parent node is always greater than or equal to its children

Algorithm (Priorit y Queue – High Level)  
Step 1: Read the number of bids N  
Step 2: Insert each bid into the priority queue  
Step 3: Extract and print the highest bid

Problem Statemen t: Determine Whether a Path Exists Between 
Two Nodes in a Graph  
Problem Description  
In a computer network or road map system, locations can be represented as nodes and 
connections between them as edges. Given a graph with multiple nodes and connections, it  is 
often necessary to determine whether a path exists between two specific nodes. Graph 
traversal algorithms such as Depth -First Search (DFS)  and Breadth -First Search (BFS)  
can be used to explore the graph and determine if the destination node can be reac hed from 
the source node.  
Input Format  
• First line: two integers N and E representing the number of nodes and edges  
• Next E lines: two integers u v representing an edge between node u and node v  
• Last line: two integers S and D representing the source node and destination node  
Output Format  
• Print “Path Exists”  if there is a path from S to D  
• Otherwise print “No Path Exists”  
Sample Input  
5 5 
0 1 
0 2 
1 3 
2 4 
3 4 
0 4 
Sample Output  
Path Exists  
Explanation  
• The graph contains 5 n odes and 5 edges  
• We need to check if a path exists from node 0 to node 4 
• Using DFS or BFS traversal, the algorithm explores neighboring nodes  
• A valid path is found: 0 → 2 → 4  
Conceptual Difference Between DFS and BFS  
• Depth -First Search (DFS)  explor es a path completely before backtracking  
• DFS uses a stack  or recursion for traversal  
• It is useful when exploring deep paths in a graph  
• Breadth -First Search (BFS)  explores nodes level by level  
• BFS uses a queue  to visit all neighbors first  
• It is us eful for finding the shortest path in unweighted graphs  
Algorithm (Graph Path Check – BFS Concept)  
Step 1: Read the number of nodes N and edges E 
Step 2: Create an adjacency list to store graph connections  
Step 3: Read each edge and store connections betwe en nodes  
Step 4: Read source node S and destination node D 
Step 5: Create a visited array to track visited nodes  
Step 6: Initialize a queue and insert the source node  
Step 7: While the queue is not empty, remove a node from the queue  
Step 8: If the removed  node is the destination, stop and report that a path exists  
Step 9: Otherwise add all unvisited neighboring nodes to the queue  
Step 10: Continue until the queue becomes empty  
Step 11: If destination is not found, report that no path exists

Problem Statement: Find the Shortest Distance from a Source

Problem Description  
In many real -world applications such as GPS navigation, network routing, and transportation 
systems, it is important to determine the shortest distance between locations. These problems 
can be modeled using a weighted graph , where nodes represent locations and edges 
represent paths with associated distances (weights).  
Dijkstra’s algorithm is a widely used algorithm for finding the shortest path from a source 
node to all other nodes in a graph with non -negative edge weights. Th e algorithm uses a 
priority queue (min -heap)  to always select the node with the smallest known distance and 
update its neighbors efficiently.  
Input Format  
• First line: two integers N and E representing the number of nodes and edges  
• Next E lines: three i ntegers u v w  representing an edge between nodes u and v with weight 
w 
• Last line: integer S representing the source node

• Print the shortest distance from the source node to all other nodes

• The graph contains 5 nodes and 6 weighted edges  
• Starting from node 0, the algorithm finds the shortest path to all other nodes  
• For example, the shorte st path to node 1 is 0 → 2 → 1  with cost 3 
Concept of Dijkstra’s Algorithm  
• The algorithm starts from the source node  
• It keeps track of the shortest known distance  to each node  
• A priority queue (min -heap)  is used to always process the node with the sm allest distance  
• For each selected node, distances to neighboring nodes are updated  
• This process continues until all nodes are processed  
Algorithm (Dijkstra's Algorithm)  
Step 1: Read the number of nodes N and edges E 
Step 2: Create an adjacency list to store graph connections with weights  
Step 3: Initialize a distance array with infinity values  
Step 4: Set the distance of the source node to 0 
Step 5: Insert the source node into a priority queue  
Step 6: While the priority queue is not empty  
• Extract the node with the smallest distance  
• For each neighbor of the node  
• Calculate a new possible distance  
• If the new distance is smaller than the current distance, update it  
• Insert the neighbor into the priority queue  
Step 7: After processing all nodes, prin t the distance array

import java.util.*;  
 
class Pair {  
    int node, dist;  
 
    Pair(int node, int dist) {  
        this.node = node;  
        this.dist = dist;  
    } 
} 
 
public class Dijkstra {  
 
    public static void main(String[] args) {  
 
        Scanner sc = new Scanner(System.in);  
 
        int n = sc.nextInt();  
        int e = sc.nextInt();  
 
        ArrayList<ArrayList<Pair>> graph = new ArrayList<>();  
 
        for(int i=0;i<n;i++)  
            graph.add(new ArrayList<>());  
 
        for(int i=0;i<e;i++){  
            int u = sc.nextInt();  
            int v = sc.ne xtInt();  
            int w = sc.nextInt();  
 
            graph.get(u).add(new Pair(v,w));  
            graph.get(v).add(new Pair(u,w));  
        } 
 
        int source = sc.nextInt();  
 
        int[] dist = new int[n];  
        Arrays.fill(dist, Integer.MAX_VALU E); 
        dist[source] = 0;  
 
        PriorityQueue<Pair> pq = new PriorityQueue<>(  
            (a,b) -> a.dist - b.dist  
        ); 
 
        pq.add(new Pair(source,0));  
 
        while(!pq.isEmpty()){  
 
            Pair p = pq.poll();  
 
            for(Pair neighbor : graph.get(p.node)){  
 
                int newDist = p.dist + neighbor.dist;  
 
                if(newDist < dist[neighbor.node]){  
                    dist[neighbor.node] = newDist;  
                    pq.add(new Pair(neighbor.node,newDist));  
                } 
            } 
        } 
 
        System.out.println("Shortest distances from source node " + source);  
 
        for(int i=0;i<n;i++)  
            System.out.println(i + " -> " + dist[i]);

    }

        int rootA = find(a);  
        int rootB = find(b);

public class DSUOptimized {  
 
    static int parent[];  
    static int rank[];  
 
    static int find(int x){  
        if(parent[x] != x)  
            parent[x] = find(parent[x]);  
        return parent[x];  
    } 
 
    static void union(int a, int b){  
 
        int rootA = find(a);  
        int rootB = find(b);  
 
        if(rootA != rootB){  
 
            if(rank[rootA] < rank[rootB])  
                parent[rootA] = rootB;  
 
            else if(rank[rootA] > rank[rootB])  
                parent[rootB] = rootA;  
 
            else{  
                parent[rootB] = rootA;  
                rank[rootA]++;  
            } 
        } 
    } 
 
    public static void main(String[] args){  
 
        Scanner sc = new Scanner(System.in);  
 
        int n = sc.nextInt();  
        int m = sc.nextInt();  
 
        parent = new int[n+1];  
        rank = new int[n+1];  
 
        for(int i =1;i<=n;i++)  
            parent[i] = i;  
 
        for(int i=0;i<m;i++){  
 
            String op = sc.next();  
 
            if(op.equals("union")){  
                int a = sc.nextInt();  
                int b = sc.nextInt();  
                union(a,b);  
            } 
 
            else if(op.equals("check")){  
                int a = sc.nextInt();  
                int b = sc.nextInt();  
 
                if(find(a) == find(b))  
                    System.out.println("Connected");  
                else 
                    System.out.println("Not Connected");  
            } 
        } 
 
        sc.close();  
    }
