Load Balancer
Real-Time Game Servers / Matchmaking Service

Render game graphics and UI locally.
Capture player inputs and send them to the server.

Question No. 15B: Machine Learning Execution Workflow (Cloud vs. Local)
For this, you will draw two separate workflows side-by-side or top-and-bottom, followed by a comparison table.
Diagram 1: Cloud-Based Execution Workflow (Google Colab)
Components:
User Device (Browser)
Internet Connection
Google Colab Environment (Jupyter Notebook interface)
Cloud Compute Resources (Virtual CPU / High-end GPUs / TPUs)
Cloud Storage (Google Drive)
Functions & Data Flow (Arrows):
User Device: Writes code and clicks "run" -> Arrow (Code/Commands) -> Internet.
Internet -> Arrow -> Google Colab Environment.
Colab Environment sends instructions -> Arrow -> Cloud CPU/GPU (where heavy computation occurs).
Cloud CPU/GPU pulls/saves data -> Two-way Arrow -> Cloud Storage (Google Drive) (where datasets and final trained models are stored).
Colab Environment sends results -> Arrow -> User Device (Program Output / Visualizations displayed).
Diagram 2: Local Execution Workflow
Components:
User Laptop Interface (IDE/Terminal)
Local RAM
Local CPU / Integrated GPU
Local Hard Drive / SSD
Functions & Data Flow (Arrows):
User Interface: Writes code -> Arrow -> Local CPU/GPU (Computation occurs here, slowly).
Local CPU/GPU loads data from -> Two-way Arrow -> Local Hard Drive/SSD (Storage).
Local CPU/GPU sends results -> Arrow -> User Interface (Program Output).
